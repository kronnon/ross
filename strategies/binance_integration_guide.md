## 4. 实时数据连接

### WebSocket 基础信息
- **端点**: `wss://stream.binance.com:9443`
- **连接限制**: 最多1024个流/连接
- **保活**: ping每20秒发送，需1分钟内回pong
- **单连接最长**: 24小时

### 常用数据流

#### K线实时推送 (核心)
```
Stream: <symbol>@kline_<interval>
```
**支持周期**: 1s, 1m, 3m, 5m, 15m, 30m, 1h, 2h, 4h, 6h, 8h, 12h, 1d, 3d, 1w, 1M

**示例**:
- `btcusdt@kline_5m` - BTC 5分钟K线
- `solusdt@kline_15m` - SOL 15分钟K线

**返回数据**:
```json
{
  "e": "kline",
  "k": {
    "o": "50000",  // 开盘价
    "c": "50200",  // 当前价
    "h": "50300",  // 最高价
    "l": "49800",  // 最低价
    "x": false     // K线是否收盘 (关键!)
  }
}
```

#### 其他数据流
| 数据流 | 用途 |
|--------|------|
| `<symbol>@depth20@100ms` | 订单簿20档 |
| `<symbol>@aggTrade` | 实时成交 |
| `<symbol>@miniTicker` | 24小时行情 |

### 订阅方式

#### 单一订阅
```
wss://stream.binance.com:9443/ws/btcusdt@kline_5m
```

#### 组合订阅 (推荐)
```
wss://stream.binance.com/stream?streams=btcusdt@kline_5m/ethusdt@kline_5m
```

## 5. 深度数据 (Order Book)

### 什么是深度数据
订单簿 = 深度数据，展示当前挂单价格分布

```
买单 (Bid)                    卖单 (Ask)
───────────────────────
价格         数量             价格        数量
50000        2.5 BTC    |   50005       1.0 BTC
49999        3.2 BTC    |   50008       0.8 BTC  
49998        1.0 BTC    |   50010       2.5 BTC

卖一: 50005    买一: 50000    价差: 5 USDT
```

### 在Ross策略中的运用

#### 1. 判断支撑/阻力强度
- 卖单墙厚(大量卖单挂起) → 阻力强，可能回落
- 卖单墙薄(几乎无阻力) → 容易突破

#### 2. 预判假突破
- 突破瞬间大量卖单砸出 → 可能假突破
- 突破后买单迅速补充 → 真突破概率高

#### 3. 简单指标
```python
# 买卖单总量比
bid_volume = sum(bids[:10].quantity)
ask_volume = sum(asks[:10].quantity)
ratio = bid_volume / ask_volume
```

### 注意
对于Ross纯价格形态策略，深度数据是**辅助工具**，非必须。核心是价格形态 + 收盘确认。

## 6. Ross 策略接入方案

### 启动流程配置

```
┌─────────────────────────────────────────────────────┐
│  1. 连接WS前，先调REST API获取历史K线                │
│  2. 写入内存缓存/数据库                              │
│  3. 连接WebSocket接收实时数据                        │
│  4. 开始交易                                         │
└─────────────────────────────────────────────────────┘
```

### 启动配置参数

| 参数名 | 说明 | 推荐值 |
|--------|------|--------|
| `INITIAL_KLINES_COUNT` | 启动时预加载K线数量 | 200 |
| `RSI_PERIOD` | RSI周期 | 14 |
| `KDJ_PERIOD` | KDJ周期 | 9 |
| `BOLL_PERIOD` | BOLL周期 | 20 |
| `TREND_KLINES_COUNT` | 判断趋势需要的K线数 | 50 |

### 各指标所需最小数据量

| 指标/形态 | 最小K线数 | 5分钟周期耗时 |
|----------|----------|--------------|
| RSI (14) | 14根 | ~70分钟 |
| KDJ (9) | 9根 | ~45分钟 |
| BOLL (20) | 20根 | ~100分钟 |
| 1-2-3形态 | 视形态 | 不确定 |

**配置建议**: 预加载200根K线可覆盖所有指标计算，启动即可交易。

### 推荐配置
| 数据流 | 用途 |
|--------|------|
| `<symbol>@kline_5m` | 5分钟K线生成信号 |
| `<symbol>@kline_15m` | 15分钟确认趋势 |
| `<symbol>@depth20` | 辅助判断支撑/阻力 |

### 关键点
- **等K线收盘** (`x: true`) 再判断信号
- **忽略跳空突破** (Gap breakout)
- **只做"穿越"入场** (非挂单)

### 示例配置 (config.json)
```json
{
  "trading": {
    "symbols": ["sol", "btc"],
    "interval": "5m",
    "initial_klines_count": 200
  },
  "indicators": {
    "rsi_period": 14,
    "kdj_period": 9,
    "boll_period": 20
  }
}
```

## 7. 数据存储架构

### 运行时数据流程

```
┌─────────────────────────────────────────────────────────┐
│                    Python进程 (内存)                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  klines_buffer: List[Kline]                     │   │
│  │  ┌────┬────┬────┬────┬────┬────┬────┐          │   │
│  │  │ K1 │ K2 │ K3 │ ...│ K98│ K99│ K100│ ←最新   │   │
│  │  └────┴────┴────┴────┴────┴────┴────┘          │   │
│  │         ↑ 策略实时读写这里                         │   │
│  └─────────────────────────────────────────────────┘   │
└───────────────────────────┬─────────────────────────────┘
                            │
                            │ 启动时读取 / 运行中不写入
                            ▼
┌─────────────────────────────────────────────────────────┐
│              MongoDB (磁盘/持久化)                       │
│  - 历史数据存储 (回测/复盘用)                            │
│  - 启动时加载最近200根到内存                             │
│  - 运行中无需实时写入                                    │
└─────────────────────────────────────────────────────────┘
```

### 内存保留K线数量

| 参数名 | 说明 | 推荐值 |
|--------|------|--------|
| `KLINES_IN_MEMORY` | 运行时内存保留K线数 | **100** |

### 为什么只需要100根？

| 用途 | 需要的K线数 |
|------|------------|
| RSI (14周期) | 14根 |
| KDJ (9周期) | 9根 |
| BOLL (20周期) | 20根 |
| 1-2-3形态识别 | 10-20根 |
| 趋势判断 | 20-50根 |

100根K线 ≈ 几KB内存占用，几乎无开销，足够覆盖所有指标和形态识别。

## 8. 两种运行模式

### 模式A: 纯内存 (推荐用于模拟交易)

```
启动: MongoDB读取200根 → 加载到Python list
运行: WebSocket新K线 → 追加到list (保留100根)
结束: 数据丢弃
```
- 优点: 简单、快速、无需维护MongoDB写入
- 适用: 快速测试、单次运行

### 模式B: 内存+MongoDB持久化 (可选)

```
启动: MongoDB读取200根 → 加载到Python list
运行: WebSocket新K线 → 追加到list + 写入MongoDB
结束: 数据保留
```
- 优点: 保留完整历史、方便复盘
- 适用: 长期运行、需要交易记录
